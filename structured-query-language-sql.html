What is SQL?

SQL is a language with a relatively simple and intuitive syntax that uses written statements (or queries) to interact with the RDBMS to perform various operations such as creating databases, tables and other objects, inserting or updating data in tables and, of course, retrieve data.

Syntax

A SQL statement typically looks like this.

SELECT [DatabaseName].[FieldName] 
FROM [DatabaseName].[TableName];
JOIN [DatabaseName].[SomeOtherTableName]
  ON [condition]
WHERE [Condition]
GROUP BY [non aggregate fields to be returned]
ORDER BY [field(s) to use for ordering, either ascending (ASC) or descending (DESC)];

In English, the statement SELECTs some data FROM some table JOINed with some other table ON some condition, and WHERE the final joined tables meet some other condition. The aggregated values (if used) are GROUPed BY the other fields selected. Finally, the results are ORDERed BY one of the field values in ASCending or DESCending order.

Not all of these parts are necessary for all statements. What you need to include depends on what you are trying to achieve. However, you will rarely have any other components than these in your statements. You can write your statements on a single line or spread them on multiple lines like I do. However, you must always respect the order of the elements. Your statement will not work if you put the GROUP BY part before the WHERE part, for instance. Also, try to remember to end your statements with a semi-colon.

Finally, keep in mind that SQL is not case sensitive, so “banana,” “BANANA,” and “BaNaNa” are the same. This also applies to the name of functions and operators. For my examples, I capitalize the functions and operators so that they can be more easily distinguished from the other elements of the statements.

In the next section, I provide examples of the different functions and operators that you may need to use in your statements. These are far from exhaustive, but should cover 95% of the basic queries that users make when working with relational databases. Administrators of databases will typically use much more functions, but that is beyond the scope of this course.

Create, modify and delete objects

The CREATE, ALTER, and DROP function can be used in SQL statements to create, modify and delete objects such as databases (schemas), tables and indices.

CREATE

The following example displays a SQL statements that create a new database called gummy_db.

CREATE DATABASE gummy_db;

The following statement creates a table called gummies that contains two columns: gummy_id, gummy_name.

CREATE TABLE gummies (
  gummy_id INT,
  gummy_name VARCHAR(45),
);

ALTER

The following statement modifies the gummies table to add a new column called gummy_description, and add a primary key (the gummy_id field).

ALTER TABLE gummies
ADD COLUMN gummy_description VARCHAR(1000),
ADD PRIMARY KEY (gummy_id);

DROP

The following statement uses the DROP function in combination with the ALTER function to remove the primary key that we added in the previous statement.

ALTER TABLE gummies
DROP COLUMN gummy_description VARCHAR(1000),
DROP PRIMARY KEY;

And we now DROP the gummies table, and the gummy_id database.

DROP TABLE gummies;
DROP DATABASE gummy_id;

Insert data in a table

The following statement creates a record in the gummies table with 1 as the gummy_id and “frogs” as the gummy_name.

INSERT INTO gummy_id (gummy_id, gummy_name)  
SELECT 1, "frogs"

Comparison operators

The comparison operators are used to compare values, the result of a comparison is always TRUE (1) or FALSE (0).

Operator Description
= Equal
> Greater than
< Less than
>= Greater than or equal to
<= Less than or equal to
<> or != Not equal to
IN Checks if value is contained in a list of values
BETWEEN … AND … Checks if value is between two values

Here is a simple example that shows how the comparison operators work. The returned value for each column is 1 if the statement is true, and 0 if it’s false.

SELECT "banana"="apple", 10 < 0, "b" BETWEEN "a" AND "c", 5 >= 5, 8 in (1,4,6,8);

Logical (boolean) operators

The logical operators are used to combine multiple conditions that must all be true for the data to be returned.

Operators Description
AND Returns 1 or TRUE when all conditions are TRUE. Returns 0 or FALSE otherwise
OR Returns 1 or TRUE when at least one of multiple conditions is TRUE. Returns 0 or FALSE otherwise
NOT Returns TRUE or 1 when the operand is FALSE or 0, and returns FALSE or 0 when the operand is TRUE or 1.

We can test the boolean operators with the following statement.

SELECT TRUE, FALSE, NOT FALSE, NOT TRUE, TRUE AND FALSE, TRUE OR FALSE;
Table 13: 1 records
TRUE FALSE NOT FALSE NOT TRUE TRUE AND FALSE TRUE OR FALSE
1 0 1 0 0 1

You’ll notice that MySQL returns 0 for FALSE and 1 for TRUE. In fact, any numeric value other than 0 is equivalent to TRUE. Here is the same example with values used in the statement instead of TRUE or FALSE.

SELECT 1, 0, NOT 0, NOT 1, 25 AND 0, 9999 OR 0;
Table 14: 1 records
1 0 NOT 0 NOT 1 25 AND 0 9999 OR 0
1 0 1 0 0 1

Let’s now try this in a query and retrieve all gummies from the gummy_db database that are blue and AND a semi-hard consistency.

SELECT gummy_id, gummy_name, gummy_description, gummy_consistency, gummy_color
FROM gummies 
WHERE gummy_color = "blue" 
  AND gummy_consistency = "semi-hard"

aggregate functions

Function Description
MIN() Returns the minimum value of a column.
MAX() Returns the maximum value of column.
SUM() Returns the sum of all values in a column.
AVG() Returns the average value of a column.
COUNT() Returns the number of rows in a table.

Let’s try the aggregate operators by querying the gummy_db database to retrieve the number of gummies in the catalogue and their minimum, maximum, and average price.

SELECT COUNT(*), MIN(gummy_price), MAX(gummy_price), AVG(gummy_price)
FROM gummies;
Table 16: 1 records
COUNT(*) MIN(gummy_price) MAX(gummy_price) AVG(gummy_price)
5 0.05 0.25 0.16

We can see that there are 5 entries in the gummies table, with prices ranging from 0.05$ to 0.25$ and an average price of 0.16$.

Often times, you will want to retrieve these aggregate values for different subsets of records (e.g. to perform group comparisons). This is where the GROUP BY function comes into play. In fact, you must include a GROUP BY in your statement whenever you use an aggregate operator, unless all of your returned values are aggregates (as in the example above). Let’s write a query that returns the number of records, and the minimum, maximum, and average price for gummies of different consistency.

SELECT gummy_consistency, COUNT(*), MIN(gummy_price), MAX(gummy_price), AVG(gummy_price)  
FROM gummies
GROUP BY gummy_consistency;
Table 17: 3 records
gummy_consistency COUNT(*) MIN(gummy_price) MAX(gummy_price) AVG(gummy_price)
Semi-hard 3 0.15 0.20 0.166667
Soft 1 0.05 0.05 0.050000
Very hard 1 0.25 0.25 0.250000

It’s important to know that MySQL will not return an error message if you forget the GROUP BY part of your statement. But will instead provide an erroneous result: the first value for gummy_constitency and the aggregate value for all of the records in the table. So remember to always examine the results of your queries (no matter what they are) to see if they are consistent with your expectations, and to not simply trust that all is fine and that the query worked and did what you wanted it to do because there is no error or warning message.

Aliases (AS)

Aliases are used to rename to rename things within your SQL statements. You can change the name of a field or table. You can even change the data type of a field. Unless you are using aliases in a CREATE or ALTER statement the Aliases do not modify your database. They are simply used for the specific statement that you are executing.

Simplifying queries

Aliases can be very useful to simplify your queries. If you look above at the statements used to provide an example of JOIN statemetns, you will notice that in the JOIN needs to include both the name of the table and the field (TableName.FieldName). SQL statements can quickly get pretty long to write (and harder to read) when writing slightly long statements. So let’s take that one of these example again and use aliases for the tables with the AS function.

SELECT *
FROM store_1 AS a -- Here I tell SQL that the new name for the table "store_1" is "a"
JOIN store_2 AS b -- I tell SQL that the new name for the table "store_2" is "b"
  ON b.id_gummy = a.id_gummy; -- So I can now write b.id_gummy = a.id_gummy instead.
Table 24: 2 records
id_gummy gummy_name id_gummy gummy_name
3 Peak-a-Chew 3 Peak-a-Chew
4 Crocodile Gumdee 4 Crocodile Gumdee

Setting column names

Another typical use of aliases is to choose a column name that is more appropriate for a display that is intended for the user. For instance, while the f field gummy_name might be useful for the database, when displayed on the website or in an invoice, we might prefer to simply use Name or some other expression. Here’s an example:

SELECT id_gummy AS ID, gummy_name AS Name 
FROM store_1;

Database views

What are views? technically they are not much more than a virtual table generated with a stored query (in some DBMS, they views are actually called “stored queries”). However, in practice they are most useful when thought of as things that your users may wish or need to see (or “view”) to obtain information or perform a certain task.

Single table view

Creating views in MySQL is pretty simple, here is an example of a single table view that simply provides the list of customers and their email address.

CREATE VIEW customers_emails AS
SELECT DISTINCT first_name, last_name, email FROM customers;

So now we can view the list of emails by retrieving records from our view.

SELECT * FROM customers_emails;
